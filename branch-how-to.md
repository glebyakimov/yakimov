# Руководство по работе с ветками в Git

## Создание ветки

### 1. Создание новой ветки
```bash
git branch <имя-ветки>
(создает ветку, но не переключает на нее)

### 2. Создание и сразу переключение на ветку
git checkout -b <имя-ветки>
или (в новых версиях Git):
git switch -c <имя-ветки>

### 3. Создание ветки из определенного коммита
git branch <имя-ветки> <хеш-коммита>

## Переключение между ветками

### 1. Переключение на существующую ветку
git checkout <имя-ветки>
или:
git switch <имя-ветки>

### 2. Переключение на предыдущую ветку
git checkout -

### 3. Просмотр всех ветокgit branch       # локальные ветки
git branch -a    # все ветки (включая удаленные)

## Отправка ветки на сервер

### 1. Отправка новой ветки в удаленный репозиторий
git push -u origin <имя-ветки>
Флаг -u устанавливает upstream (связь между локальной и удаленной веткой)

### 2. Отправка изменений в существующую удаленную веткуgit push
(работает если upstream уже установлен)

### 3. Принудительная отправка (перезапись истории)git push -f
(используйте с осторожностью!)
